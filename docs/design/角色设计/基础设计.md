# 角色基础设计

## 当前定位

- 当前角色正式定名为 `士兵`，作为三国杀 mod 的基础模板角色。
- 立绘、战斗场景、商店场景、篝火场景后续再补。
- 当前内部类型名收敛为 `SoldierCharacter`，后续不建议再频繁改动。

## 当前实现约束

- 资源暂时复用原版 `ironclad` 作为占位角色资源来源。
- 不启用自定义 `Epoch` / `Timeline`，`RequiresEpochAndTimeline = false`。
- 不创建自定义角色场景、能量表盘场景、商店场景、篝火场景，等角色视觉方案定稿后再补。
- 当前与后续新增的 mod 卡牌内容，默认都归属 `士兵` 卡池。
- 装备槽与装备牌使用权限当前绑定 `士兵`；非士兵不能使用装备牌。

## 资源占位规范

当前阶段所有角色资源都允许先写成“占位符需求”，不要求立刻提供实物素材。

建议每项资源都按下面格式描述：

- `资源状态`：占位 / 已提供 / 已接入
- `资源类型`：立绘 / 战斗图 / 角色选择图 / 能量图标 / 音效 / BGM / 商店场景 / 篝火场景
- `资源用途`：说明触发界面或使用节点
- `占位文件名`：即使暂时没有文件，也先约定目标文件名
- `最终格式`：`png` / `svg` / `ogg` / `wav` / `tscn`
- `备注`：尺寸、时长、循环、透明背景等要求

示例：

- `资源状态`：占位
- `资源类型`：角色选择图
- `资源用途`：角色选择界面头像
- `占位文件名`：`char_select_soldier.png`
- `最终格式`：`png`
- `备注`：透明背景，后续替换当前原版占位资源

音频也按同样方式记录：

- `资源状态`：占位
- `资源类型`：攻击音效
- `资源用途`：角色普通攻击时播放
- `占位文件名`：`soldier_attack.ogg`
- `最终格式`：`ogg`
- `备注`：短音效，可重复触发

## 士兵专属机制

- `士兵` 拥有 4 个装备槽位：武器、防具、坐骑、宝物。
- 装备槽不是通用角色机制，而是 `士兵` 的角色专属机制。
- 非 `士兵` 角色即使通过抽牌、生成、复制或其他效果获得装备牌，也不能打出并装备。
- 当前代码已经按这个规则限制了装备牌的打出、装备和装备状态查询入口。

## 与身份牌的关系

- `IdentityBadgeRelic` 已作为该角色的初始遗物挂入 starter 注册。
- 遗物当前归属 `士兵` 的遗物池。
- 当前项目默认把已实现内容视为 `士兵` 体系内容。
