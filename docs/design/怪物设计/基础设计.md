# 怪物设计

当前仓库尚未沉淀正式怪物清单；这一阶段先固定怪物设计的文档格式，避免后续在教程接口、遭遇结构和资源占位上反复返工。

## 当前技术方案

- 使用教程主线 [RitsuLib/11 - 添加新怪物](/C:/workspace/python/game/SlayTheSpire2ModdingTutorials/RitsuLib/11%20-%20添加新怪物/README.md) 作为实现参考。
- 单个怪物统一继承 `ModMonsterTemplate`。
- 单个遭遇统一继承 `ModEncounterTemplate`。
- 怪物需要同时补齐：
  - `Scripts/Monsters/<怪物名>.cs`
  - `MyFirstStS2Mod/localization/zhs/monsters.json`
  - 怪物视觉场景或占位场景
- 遭遇需要同时补齐：
  - `Scripts/Encounters/<遭遇名>.cs`
  - `MyFirstStS2Mod/localization/zhs/encounters.json`
  - 多怪遭遇使用的遭遇场景或占位场景
  - `Entry.Init()` 里的遭遇注册

## 设计分层

怪物设计建议拆成两层：

- `怪物本体`
  - 名称
  - 类型：普通怪 / 精英 / Boss
  - 所属层数
  - 血量范围
  - 入场自带能力
  - 每个行动意图
  - 出招规则
- `遭遇设计`
  - 单怪 / 多怪
  - 所属层数
  - 房间类型：Monster / Elite / Boss
  - 遭遇标题
  - 怪物组合
  - 是否弱怪池
  - 是否需要专用遭遇场景

## 资源占位规范

当前阶段所有怪物资源都允许先写成“占位符需求”，不要求立刻提供实物素材。

建议每项资源都按下面格式描述：

- `资源状态`：占位 / 已提供 / 已接入
- `资源类型`：战斗图 / Spine 场景 / 遭遇场景 / 音效 / 台词音效
- `资源用途`：说明触发界面或使用节点
- `占位文件名`：即使暂时没有文件，也先约定目标文件名
- `最终格式`：`png` / `tscn` / `ogg` / `wav`
- `备注`：尺寸、是否循环、是否多段动画、是否透明背景

## 单个怪物设计最小字段

每个怪物设计稿建议至少补齐：

- `怪物名`
- `正式类型名`
- `类型`
- `所属层数`
- `资源状态`
- `低进阶生命`
- `高进阶生命`
- `入场效果`
- `行动1`
- `行动2`
- `出招规则`
- `关联遭遇`

## 单个行动意图最小字段

每个行动建议固定写法：

- `行动名`
- `意图显示`
  - 单攻 / 多段攻击 / 防御 / Buff / Debuff / 召唤 / 治疗 / 混合
- `低进阶数值`
- `高进阶数值`
- `实际效果`
- `补充说明`

## 遭遇设计最小字段

每个遭遇设计稿建议至少补齐：

- `遭遇名`
- `正式类型名`
- `所属层数`
- `房间类型`
- `弱怪池`
- `怪物组合`
- `槽位要求`
- `资源状态`
- `结算文案`

## 当前约束

- 怪物图片、遭遇场景、音效当前都允许先占位。
- 没有明确设计稿时，不提前发明怪物行动和 AI。
- 特殊战斗、阶段转换、召唤物链条、专用背景等高耦合内容，只有在设计稿明确后再落代码。
